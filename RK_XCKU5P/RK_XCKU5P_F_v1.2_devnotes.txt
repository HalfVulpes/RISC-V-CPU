3. MEMORY
--------------------------------------------------------------------------------
  DDR4 SDRAM (on-board):
    Chips         : 2x Micron MT40A512M16LY-062E
    Organization  : 512 Mrows x 16-bit per chip = 1 GB each, 2 GB total
    Bus Width     : 32-bit (2 chips in parallel, x16 each)
    Max Speed     : DDR4-3200 (062E speed grade) -- run at DDR4-2666 here
    FPGA Banks    : HP Bank 64 (DQ[0:15], DM/DQS for bytes 0-1)
                    HP Bank 65 (addr/ctrl/clock and DQ[16:31], DM/DQS for
                                bytes 2-3 -- yes, DQs split across banks)

  [CRITICAL] MIG IP CONFIGURATION
    The MT40A512M16LY-062E part is NOT in the Vivado 2023.2 catalog.
    Select "MT40A512M16HA-075E" instead -- the HA-075E timing database in
    MIG matches the actual LY chips at DDR4-2666. Do NOT use "MT40A512M16LY-075":
    the LY-075 entry targets a different internal silicon generation and will
    fail DQS_GATE calibration on this board.

    Verified MIG settings (from KU5P_DEMO/06_DDR_AXI/ddr4_0.xci):
       C0.DDR4_MemoryPart            MT40A512M16HA-075E
       C0.DDR4_MemoryVoltage         1.2 V
       C0.DDR4_MemoryType            Components
       C0.DDR4_Slot                  Single
       C0.DDR4_InputClockPeriod      5000 ps   (200 MHz input)
       C0.DDR4_TimePeriod            750  ps   (DDR4-2666)
       C0.DDR4_PhyClockRatio         4:1
       C0.DDR4_DataWidth             32
       C0.DDR4_DataMask              DM_NO_DBI
       C0.DDR4_CasLatency            19
       C0.DDR4_CasWriteLatency       14
       C0.DDR4_ChipSelect            true
       C0.DDR4_OutputDriverImpedenceControl  RZQ/7
       C0.DDR4_OnDieTermination      RZQ/6

  DDR4 Pinout (authoritative -- from factory MIG XDC)
    Controls:
       sys_clk_p  = T24      sys_clk_n = U24   (DIFF_SSTL12, shared with user PLL)
       act_n      = Y21
       cs_n[0]    = AE16
       cke[0]     = AE18
       odt[0]     = AE26
       reset_n    = AC16
       ba[0]      = AE17     ba[1]     = AF17
       bg[0]      = AD16     (x16 parts: no bg[1])
       ck_t[0]    = AA22     ck_c[0]   = AB22
    Address [0:16]:
       adr[0]  = AE22  adr[1]  = AF22  adr[2]  = AD23  adr[3]  = AE23
       adr[4]  = AC22  adr[5]  = AC23  adr[6]  = AB21  adr[7]  = AC21
       adr[8]  = AF20  adr[9]  = AD20  adr[10] = AE20  adr[11] = AC19
       adr[12] = AD19  adr[13] = AF18  adr[14] = AF19  adr[15] = AC18
       adr[16] = AD18
    DQS pairs (T/C):
       dqs[0]  = AC26/AD26   dqs[1]  = AB17/AC17
       dqs[2]  = V21 /V22    dqs[3]  = W25 /W26
    Data mask:
       dm_n[0] = AE25  dm_n[1] = Y20   dm_n[2] = U19   dm_n[3] = Y22
    Data byte 0 (Bank 64, DQ[0:7]):
       AB25 AB26 AF24 AF25 AD24 AD25 AB24 AC24
    Data byte 1 (Bank 64, DQ[8:15]):
       AA19 AB19 AA20 AB20 Y17  AA17 Y18  AA18
    Data byte 2 (Bank 65, DQ[16:23]):
       U21  U22  T20  U20  T22  T23  W19  W20
    Data byte 3 (Bank 65, DQ[24:31]):
       Y25  Y26  AA24 AA25 V23  W23  V24  W24

  QSPI Flash:
    Chip          : MX25U51245GZ4I00 (Macronix)
    Capacity      : 512 Mb = 64 MB
    Voltage       : 1.8 V CMOS, x4 SPI
    Usage         : Stores FPGA configuration bitstream (+ optional user data)
    FPGA Bank     : Bank 65 (QSPI0_* signals; configured via BITSTREAM settings)

4. CLOCK SOURCES
--------------------------------------------------------------------------------
  Primary System Clock (POPULATED):
    Component     : SG3225VAN 200.000000M-KEGA3  (reference Y2)
    Frequency     : 200.000 MHz, differential
    Pins          : T24 (SYS_CLK_P) / U24 (SYS_CLK_N)
    FPGA Bank     : 65 (HP, 1.2 V)
    IOSTANDARD    : DIFF_SSTL12
    Usage         : DDR4 reference clock AND general user logic source.
                    There is no other populated clock in the HP-bank area.

  GT Reference Clock for QSFP28 (POPULATED):
    Component     : SJT9120AI-2D3-33E15S.25000  (reference Y3)
    Frequency     : 156.25 MHz, differential
    Pins          : V7 (GT_CLK156P25_P) / V6 (GT_CLK156P25_N)
    FPGA Bank     : 225 (GTY MGTREFCLK0)
    Usage         : QSFP28 100G optical transceivers

  Reserved Single-Ended Clock (NOT POPULATED by default):
    Component     : Y4  (marked "Reserved - Not Soldered")
    Pin           : PL_CLK (HD bank)
    Note          : Populate if you need an HD-bank clock source

5. PERIPHERAL INTERFACES
--------------------------------------------------------------------------------

  5.1 GIGABIT ETHERNET
    PHY Chip      : Realtek RTL8211F-CG
    Standard      : 10 / 100 / 1000 Mbps, RGMII
    FPGA Bank     : 66 (HP, 1.8 V)
    IOSTANDARD    : LVCMOS18
    Default IP    : 192.168.1.10 (factory test image)
    Host Setup    : Set host PC to 192.168.1.102 / 255.255.255.0
    Signals       : PHY1_RXCK, PHY1_RXCTL, PHY1_RXD[0:3]
                    PHY1_TXCK, PHY1_TXCTL, PHY1_TXD[0:3]
                    PHY1_MDC, PHY1_MDIO
    [IMPORTANT]   : PHYRSTB and INTB are NOT routed to FPGA GPIO on this
                    board -- they connect only to on-board pull-up resistors.
                    Do not attempt to drive phy_reset / phy_int from the FPGA.

  5.2 UART + JTAG (on-board FT2232HQ)
    Chip          : FTDI FT2232HQ  (dual-channel USB to UART/JTAG)
    Connector     : USB Type-C (single cable provides both channels)
    Channel A     : JTAG via MPSSE (download/debug)
                    * Does NOT enumerate as /dev/ttyUSB* on Linux.
                    * Accessible via Vivado hw_server on tcp:3121.
                    * Max JTAG speed ~30 Mb/s.
    Channel B     : UART (serial console)
                    * Enumerates as /dev/ttyUSB1 on Linux.
                    * 115200 8N1 for factory images.
    FPGA Bank     : 84 (HD, 3.3 V), IOSTANDARD LVCMOS33
    Pins          : AD13 = uart_rx (FPGA input from FT2232 TXD)
                    AC14 = uart_tx (FPGA output to FT2232 RXD)

  5.3 SD CARD (MicroSD)
    Modes         : 4-bit SD mode and 1-bit SPI mode
    FPGA Bank     : 84 (HD, 3.3 V)
    IOSTANDARD    : LVCMOS33
    Pins          : Y16 = SD_CD   Y15 = SD_CLK   AA15 = SD_CMD
                    AB14 = SD_D0  AA14 = SD_D1   AB16 = SD_D2  AB15 = SD_D3
                    (SD_D0 has an on-board pull-up resistor)
    SPI aliases   : SD_D0 = MISO     SD_CMD = MOSI     SD_D3 = NCS
    [IMPORTANT]   : There is NO dedicated sd_reset / sd_power line routed to
                    the FPGA. Cards tested: MicroSD 2.0, 8 - 32 GB.

  5.4 QSFP28 OPTICAL
    Port count    : 1x QSFP28 (accepts 100G optical module)
    GT Bank       : 225 (GTY, 4 lanes, up to 25 Gb/s per lane = 100 Gb/s)
    Ref clock     : 156.25 MHz (V7/V6, MGTREFCLK0_225)
    Control Bank  : 84 (HD, 3.3 V), IOSTANDARD LVCMOS33
    Control pins  : INTL=Y13     LPMODE=W14    MODPRSL=AA13
                    MODSELL=W13  RESETL=W12    SCL=AE15     SDA=AE13
    Tested module : Intel CWDM4 100G + LC single-mode fiber jumper

  5.5 PCIe 3.0 x4
    Connector     : PCIe x8 slot (electrical x4)
    GT Bank       : 224 (GTY, 4 lanes)
    Lane speed    : up to 8 Gb/s per lane
    Ref clock     : AB7 (P) / AB6 (N)  --  MGTREFCLK0_224
    Reset pin     : T19 (Bank 65, HP 1.2 V, LVCMOS12, active LOW)
    Power supply  : Board can be powered via PCIe 12 V rail; when doing so
                    the physical power button is disabled.

  5.6 DDR4  (see Section 3 for full specs + MIG config + pinout)
    IP required   : UltraScale+ DDR4 IP (not "MIG 7 Series")
    MIG settings  : Memory part  = MT40A512M16HA-075E  (NOT -LY-075)
                    Speed        = DDR4-2666
                    DataWidth    = 32

  5.7 MIPI CSI-2 CAMERA
    Connector     : J3, 22-pin, 4-lane MIPI D-PHY (CSI-2)
    Data lanes    : Bank 65 (HP 1.2 V) -- requires MIPI RX subsystem IP
    MIPI pins     : CLK_P = U19   CLK_N = V19
                    LAN0_P = T22  LAN0_N = T23
                    LAN1_P = U21  LAN1_N = U22
                    LAN2_P = T20  LAN2_N = U20
                    LAN3_P = V21  LAN3_N = V22
    [CRITICAL]    : MIPI data-lane pins overlap with DDR4 Byte 2/3 DQ/DQS.
                    This board CANNOT use MIPI and DDR4 at the same time.
    Camera ctrl   : Bank 66 (HP 1.8 V), IOSTANDARD LVCMOS18
                    CAM_CLK = K21   CAM_RST = J21   CAM_PWDN = M21
                    CAM_SCL = J25   CAM_SDA = J26
    Tested camera : Zhengdian Atom ATK-MCIMX415 (IMX415, 4K60 via FH1159 board)
    IMX415 note   : Since V0.2 firmware, the camera uses the on-board oscillator
                    as its clock source. Do NOT route an FPGA-generated clock
                    to the camera; the previously required resistor mod is
                    no longer needed.

  5.8 FMC HPC CONNECTOR
    Standard      : VITA 57.1 FMC HPC
    LA pairs      : 34 differential pairs (LA00-LA33)
    DP lanes      : 8 GTY lanes (Bank 226: DP0-3, Bank 227: DP4-7)
    GT speed      : up to 25 Gb/s per DP lane
    GT clocks     : GBTCLK0 M2C (P7/P6), GBTCLK1 M2C (K7/K6)
    LA banks      : 66 (LA00-16) + 67 (LA17-33), both HP
    LA voltage    : VADJ1 (default 1.8 V, adjustable to 1.2 V; see Section 7)
    IOSTANDARD    : LVDS for differential pairs; LVCMOS18 for single-ended use
    I2C / PWRGD   : Bank 86 (HD 3.3 V) -- F10=SCL, F9=SDA, G10=PWRGD
    Trace lengths : All FMC LA signals matched at ~2137 mil (< 0.3 mil tolerance)

  5.9 40-PIN EXPANSION IO (J1)
    Connector     : J1, 2.54 mm pitch, HDR2X20
    Signals       : 17 differential pairs (IO1_N/P through IO17_N/P)
    Power pins    : Pin 2 = +5 V     Pins 39/40 = +3.3 V     Pins 37/38 = GND
    Banks         : 86 (IO1-IO6) + 87 (IO7-IO17), both HD
    Voltage       : 3.3 V FIXED -- CANNOT be changed
    IOSTANDARD    : LVCMOS33  (or LVDS_25 for differential)
    Trace lengths : All IO pairs length-matched at 2880.464 mil
    Compatibility : Works with Heijin (Hei Jin) expansion modules

  5.10 USER LEDs (x4, active LOW)
    Bank   : 86 (HD 3.3 V), IOSTANDARD LVCMOS33
    Logic  : Output LOW = LED ON (green, driven via MMBT3904 NPN transistor)
    Pins   : LED1=H9   LED2=J9   LED3=G11   LED4=H11
    Also   : FPGA DONE LED (white) driven by FPGA config hardware (not user IO)
             Power LED (green) always on when board is powered

  5.11 USER KEYS (x4, active LOW)
    Bank   : 86 (HD 3.3 V), IOSTANDARD LVCMOS33
    Logic  : Input LOW = key pressed (4.7 kΩ pull-up, 100 nF debounce cap)
    Pins   : KEY1=K9  KEY2=K10  KEY3=J10  KEY4=J11
    Note   : KEY1 (K9) is reused as sys_rst_n in most factory demos

  5.12 FAN
    Type   : 2-wire fan attached to the FPGA heatsink, always on (no PWM)
    Bank   : 86 (HD 3.3 V), IOSTANDARD LVCMOS33
    Pin    : G9  (drives an AO3400A N-FET with SS54 flyback diode)

6. POWER SUPPLY
--------------------------------------------------------------------------------
  Input Voltage : 12 V DC  (acceptable range: 10 V to 14 V)
  Input Current : 3 A minimum recommended (~36 W at full load)
  Connector     : Barrel jack + power button switch
  Alt. Power    : PCIe slot 12 V (power button inactive in this mode)
  Protection    : Overcurrent, reverse polarity, TVS overvoltage clamping

  Power-On Sequence (automatic on this board):
    VCCINT  ->  1.8 V  ->  3.3 V  ->  ADJ/MGTAVCC  ->  MGTAVCCAUX  ->  MGTAVTT

  Internal Rails:
    0.85 V  (35 A)  -- VCCINT (FPGA core)
    0.9  V  (6 A)   -- MGTAVTT
    1.2  V  (6 A)   -- DDR_1V2 (DDR4 memory + FPGA HP banks 64/65)
    1.8  V  (3 A)   -- VADJ1 (FMC + HP bank 66/67; default, adjustable)
    1.8  V  (1.5 A) -- PL_MGTAVCCAUX / various logic rails
    1.3  V          -- MGTAVCC
    3.3  V          -- VCC_3V3 (UART, SD, LEDs, keys, HD banks 84/86/87)
    VCC_ADC         -- FPGA ADC supply (filtered from 1.8 V)

  FMC Voltage Adjustment (VADJ1):
    Default : 1.8 V
    Change  : Modify DCDC FB resistor marked "RA" on the PCB.
              Replace RA with 1 kΩ to set VADJ1 to 1.2 V.
    [CRITICAL]  Change RA BEFORE inserting a 1.2 V FMC card; wrong voltage
                can damage the card.

7. IMPORTANT WARNINGS
--------------------------------------------------------------------------------

  [CRITICAL] Vivado version: use 2023.2 only. Vivado 2024.x or newer will
             permanently lock the MX25U51245G QSPI flash chip.

  [CRITICAL] Banks 86/87 (40-pin IO) are FIXED at 3.3 V. Do not apply signals
             above 3.3 V.

  [CRITICAL] HP banks (64, 65, 66, 67) operate at 1.2 V (64/65) or 1.8 V (66/67).
             Do not apply 3.3 V signals to HP-bank pins.

  [CRITICAL] FMC VADJ1 default is 1.8 V. If using a 1.2 V FMC card, change
             resistor RA BEFORE inserting the card.

  [CRITICAL] GTY/GT transceiver pins (PCIe, QSFP28, FMC DP) must be configured
             via the appropriate Xilinx IP wizard. Do NOT set PACKAGE_PIN or
             IOSTANDARD on GT pins manually.

  [CRITICAL] DDR4 memory part: select "MT40A512M16HA-075E" in MIG, NOT
             "MT40A512M16LY-075". The LY-075 timing model does not match the
             actual LY chips on this board and will fail DQS_GATE calibration.

  [CRITICAL] MIPI data lanes (T20/T22/T23/U19-U22/V21/V22) OVERLAP with DDR4
             Byte 2/3 DQ/DQS pins. A design cannot use both peripherals.

  [WARNING]  RTL8211F PHY reset (PHYRSTB) and interrupt (INTB) are NOT wired
             to FPGA GPIO -- they use on-board pull-ups only.

  [WARNING]  MIPI CSI-2 pins require the Xilinx MIPI RX subsystem IP; do not
             set IOSTANDARD manually on MIPI lane pins.

  [WARNING]  LEDs are active-LOW: output '0' turns a LED ON.

  [WARNING]  Keys are active-LOW: a read of '0' means the key is pressed.

  [INFO]     Factory QSPI flash already contains a test bitstream.

  [INFO]     When powered via the PCIe slot, the physical power button is
             inactive; the board is on whenever the slot supplies 12 V.

  [INFO]     Vivado ILA (Integrated Logic Analyzer) works over the FT2232HQ
             JTAG interface on Channel A.

8. VIVADO PROJECT SETUP
--------------------------------------------------------------------------------
  Target Part   : xcku5p-ffvb676-2-i
  Design Flow   : RTL Project or IP Integrator (both supported)
  Base XDC      : RK_XCKU5P_F_v1.2_pinconstraint.xdc (this directory)

                    BITSTREAM.GENERAL.COMPRESS    TRUE
                    BITSTREAM.CONFIG.CONFIGRATE   63.8
                    BITSTREAM.CONFIG.SPI_BUSWIDTH 4
                    CONFIG_VOLTAGE                1.8
                    CONFIG_MODE                   SPIx4
                    CFGBVS                        GND

  IP Cores needed for full-feature use:
    - UltraScale+ DDR4 IP        (DDR4 memory controller)
    - Xilinx PCIe IP             (UltraScale+ Integrated Block for PCIe)
    - GT Wizard                  (QSFP28 / FMC DP lanes)
    - MIPI CSI-2 Receiver        (camera input)
    - Clocking Wizard            (user PLL/MMCM from 200 MHz sys_clk)

  Recommended Vivado version: 2023.2  (factory demos shipped in 2023.1)
