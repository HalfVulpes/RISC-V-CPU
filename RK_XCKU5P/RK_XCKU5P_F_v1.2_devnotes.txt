--------------------------------------------------------------------------------

5. PERIPHERAL INTERFACES
--------------------------------------------------------------------------------

  5.1 GIGABIT ETHERNET
    PHY Chip      : Realtek RTL8211F-CG
    Standard      : 10 / 100 / 1000 Mbps, RGMII
    FPGA Bank     : 66 (HP, 1.8 V)
    IOSTANDARD    : LVCMOS18
    Default IP    : 192.168.1.10 (factory test image)
    Host Setup    : Set host PC to 192.168.1.102 / 255.255.255.0
    Signals       : PHY1_RXCK, PHY1_RXCTL, PHY1_RXD[0:3]
                    PHY1_TXCK, PHY1_TXCTL, PHY1_TXD[0:3]
                    PHY1_MDC, PHY1_MDIO

  5.2 UART + JTAG (on-board programmer)
    Chip          : FTDI FT2232HQ  (dual channel USB to UART/JTAG)
    Connector     : USB Type-C
    Channel A     : JTAG (download/debug)  -- max 30 Mb/s
    Channel B     : UART (serial console)
    FPGA Bank     : 84 (HD, 3.3 V)
    IOSTANDARD    : LVCMOS33
    Pins          : AD13 = UART_RX,  AC14 = UART_TX

  5.3 SD CARD (MicroSD)
    Modes         : SPI mode and SD mode
    FPGA Bank     : 84 (HD, 3.3 V)
    IOSTANDARD    : LVCMOS33
    Pins          : Y16=SD_CD, Y15=SD_CLK, AA15=SD_CMD
                    AB14=SD_D0, AA14=SD_D1, AB16=SD_D2, AB15=SD_D3
    SPI aliases   : SD_D0=MISO (has pull-up), SD_CMD=MOSI, SD_D3=NCS
    Cards tested  : MicroSD 2.0, 8 GB - 32 GB

  5.4 QSFP28 OPTICAL
    Port count    : 1x QSFP28 (accepts 100G optical module)
    GT Bank       : 225 (GTY, 4 lanes, up to 25 Gb/s per lane = 100 Gb/s)
    Ref clock     : 156.25 MHz (V7/V6, MGTREFCLK0_225)
    Control Bank  : 84 (HD, 3.3 V), IOSTANDARD LVCMOS33
    Control pins  : INTL=Y13, LPMODE=W14, MODPRSL=AA13
                    MODSELL=W13, RESETL=W12, SCL=AE15, SDA=AE13
    Tested module : Intel CWDM4 100G + LC fiber (LC single-mode jumper)

  5.5 PCIe 3.0 x4
    Connector     : PCIe x8 slot (electrical x4)
    GT Bank       : 224 (GTY, 4 lanes)
    Lane speed    : up to 8 Gb/s per lane
    Ref clock     : AB7 (P) / AB6 (N) -- MGTREFCLK0_224
    Reset pin     : T19 (Bank 65, HP 1.2 V) -- LVCMOS12, active LOW
    Power supply  : Can power board via PCIe slot (12 V) -- power button inactive

  5.6 DDR4  (see Section 3 for specs)
    IP required   : Xilinx MIG (Memory Interface Generator)
    MIG settings  : Part=MT40A512M16LY-062E, 32-bit, Speed=2666

  5.7 MIPI CSI-2 CAMERA
    Connector     : J3, 22-pin, 4-lane MIPI D-PHY (CSI-2)
    Data lanes    : Bank 65 (HP 1.2 V) -- requires MIPI RX IP core
    MIPI pins     : CLK_P=U19, CLK_N=V19
                    LAN0_P=T22, LAN0_N=T23
                    LAN1_P=U21, LAN1_N=U22
                    LAN2_P=T20, LAN2_N=U20
                    LAN3_P=V21, LAN3_N=V22
    Camera ctrl   : Bank 66 (HP 1.8 V), IOSTANDARD LVCMOS18
                    CAM_CLK=K21, CAM_RST=J21, CAM_PWDN=M21
                    CAM_SCL=J25, CAM_SDA=J26
    Tested camera : Zhengdian Atom ATK-MCIMX415 (IMX415, 4K60 via FH1159 board)
    IMPORTANT (V0.2): The IMX415 camera BUG fix -- do NOT drive a clock from
                    FPGA IO to camera; use the on-board oscillator clock only.
                    (Resistor mod previously required is NO LONGER needed with
                    current firmware as the bug has been fixed in software.)

  5.8 FMC HPC CONNECTOR
    Standard      : VITA 57.1 FMC HPC
    LA pairs      : 34 differential pairs (LA00-LA33)
    DP lanes      : 8 GTY lanes (Bank 226: DP0-3, Bank 227: DP4-7)
    GT speed      : up to 25 Gb/s per DP lane
    GT clocks     : GBTCLK0 M2C (P7/P6), GBTCLK1 M2C (K7/K6)
    LA bank       : 66 (LA00-16, HP) + 67 (LA17-33, HP)
    LA voltage    : 1.8 V default (VADJ1, adjustable -- see Section 7 WARNING)
    IOSTANDARD    : LVDS for differential pairs; LVCMOS18 for single-ended use
    I2C / PWRGD   : Bank 86 (HD 3.3 V) -- F10=SCL, F9=SDA, G10=PWRGD
    Trace lengths : All FMC LA signals matched at ~2137 mil (see equal-length XLS)

  5.9 40-PIN EXPANSION IO
    Connector     : J1, 2.54 mm pitch, HDR2X20 (40 pins)
    Signals       : 17 differential pairs (IO1_N/P through IO17_N/P)
    Power pins    : Pin 2 = +5 V, Pins 39/40 = +3.3 V, Pins 37/38 = GND
    Banks         : 86 (IO1-IO6) + 87 (IO7-IO17)
    Voltage       : 3.3 V FIXED -- CANNOT be changed
    IOSTANDARD    : LVCMOS33  (single-ended) or LVDS_25 (differential)
    Trace lengths : All IO signals matched at 2880.464 mil
    Compatibility : Compatible with Heijin (Hei Jin) expansion modules

  5.10 USER LEDs  (x4, active LOW)
    Bank   : 86 (HD 3.3 V), IOSTANDARD LVCMOS33
    Logic  : Output LOW = LED ON (green LED, driven via MMBT3904 NPN transistor)
    LED1=H9, LED2=J9, LED3=G11, LED4=H11
    Also:  FPGA DONE LED (white, driven by FPGA config done signal -- not user IO)
           Power LED (green, always on when board is powered)

  5.11 USER KEYS  (x4, active LOW)
    Bank   : 86 (HD 3.3 V), IOSTANDARD LVCMOS33
    Logic  : Input LOW = key pressed (4.7 K pull-up to 3.3 V + 100 nF debounce cap)
    KEY1=K9, KEY2=K10, KEY3=J10, KEY4=J11
    Note   : KEY1 (K9) also used as system reset in factory demos

  5.12 FAN
    Type   : 2-wire fan with heatsink on FPGA -- always on, no PWM speed control
    Bank   : 86 (HD 3.3 V), IOSTANDARD LVCMOS33
    Pin    : G9 (FAN control via AO3400A N-FET + SS54 flyback diode)

6. POWER SUPPLY
--------------------------------------------------------------------------------
  Input Voltage : 12 V DC  (acceptable range: 10 V to 14 V)
  Input Current : 3 A minimum recommended (12V x 3A = 36W)
  Connector     : Barrel jack + power button switch
  Alt. Power    : PCIe slot 12 V (power button inactive when powered via PCIe)
  Protection    : Overcurrent, reverse polarity, TVS overvoltage protection

  Power-On Sequence (must be followed by hardware -- automatic on this board):
    VCCINT  ->  1.8 V  ->  3.3 V  ->  ADJ/MGTAVCC  ->  MGTAVCCAUX  ->  MGTAVTT

  Internal Rails:
    0.85 V  (35 A)  -- VCCINT (FPGA core)
    0.9  V  (6 A)   -- MGTAVTT
    1.2  V  (6 A)   -- DDR_1V2 (DDR4 memory + FPGA HP bank 64/65)
    1.8  V  (3 A)   -- VADJ1 (FMC + HP bank 66/67 -- default, adjustable)
    1.8  V  (1.5 A) -- PL_MGTAVCCAUX / various logic rails
    1.3  V          -- MGTAVCC
    3.3  V          -- VCC_3V3 (UART, SD, LEDs, keys, HD banks 84/86/87)
    VCC_ADC         -- FPGA ADC supply (filtered from 1.8 V)

  FMC Voltage Adjustment (VADJ1):
    Default : 1.8 V
    Change  : Modify DCDC chip FB resistor marked "RA" on PCB
              To set 1.2 V: replace RA with 1 K ohm resistor

7. IMPORTANT WARNINGS & NOTES
--------------------------------------------------------------------------------

  [CRITICAL] Bank 86/87 (40-PIN IO) voltage is FIXED at 3.3 V.
             Do NOT apply voltages above 3.3 V on these pins.

  [CRITICAL] HP banks (64, 65, 66, 67) operate at 1.2-1.8 V.
             Do NOT apply 3.3 V signals to HP bank pins.

  [CRITICAL] FMC VADJ1 defaults to 1.8 V. If connecting an FMC card that
             requires 1.2 V, change resistor RA BEFORE inserting the card.
             Failure to do so may damage the FMC card.

  [CRITICAL] GTY/GT transceiver pins (PCIe, QSFP28, FMC DP) must be configured
             via the appropriate Xilinx IP wizard (PCIe IP, GT Wizard).
             DO NOT set PACKAGE_PIN or IOSTANDARD on GT pins manually.

  [CRITICAL] DDR4 pin constraints must be generated by Xilinx MIG IP.
             Do NOT add manual DDR4 constraints -- MIG sets all DDR IOSTANDARD,
             drive strength, and termination properties automatically.

  [WARNING]  MIPI CSI-2 pins require a Xilinx MIPI RX subsystem IP.
             Do NOT set IOSTANDARD on MIPI data lane pins manually.

  [WARNING]  MIPI camera (IMX415): Use on-board oscillator as camera clock.
             Do NOT drive camera clock from FPGA IO pin (known hardware bug,
             software workaround exists since V0.2 -- resistor mod not needed).

  [WARNING]  LEDs are active LOW: output '0' turns LED ON.

  [WARNING]  Keys are active LOW: read '0' when key is pressed.

  [INFO]     Factory QSPI Flash already contains a test bitstream.
             To update: follow "Chapter 3 - Flash Factory Bit" in user manual.

  [INFO]     When powering via PCIe slot, the physical power button has no effect.

  [INFO]     Vivado ILA (Integrated Logic Analyzer) can be used for on-board
             debugging via the FT2232HQ JTAG interface.

8. VIVADO PROJECT SETUP
--------------------------------------------------------------------------------
  Target Part   : xcku5p-ffvb676-2-i
  Design Flow   : RTL Project (not IP Integrator required, but supported)
  Constraint    : Use RK_XCKU5P_F_v1.2_pinconstraint.xdc as the base XDC
  Bitstream     : Always set BITSTREAM.GENERAL.COMPRESS TRUE
                  and BITSTREAM.CONFIG.CONFIGRATE 63.8 for faster boot
  IP Cores needed for full feature use:
    - MIG 7 Series / UltraScale+ DDR4 IP  (for DDR4 memory)
    - Xilinx PCIe IP (UltraScale+ Integrated Block for PCIe)
    - GT Wizard (for QSFP28 / FMC DP transceiver lanes)
    - MIPI CSI-2 Receiver Subsystem  (for camera input)
    - Clocking Wizard (for PLL/MMCM from 200 MHz sys clock)
  Recommended Vivado version: 2023.1  (used in all factory demo projects)
